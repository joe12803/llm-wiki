# GitNexus (GitNexus MCP)

GitNexus 是一个开源项目，旨在通过提供“上帝视角”和“代码地图”来增强 AI 编程工具（如 Codex、Claude Code、Cursor）对大型代码库的理解能力。它通过将代码库索引为知识图谱，解决了 AI 编程助手缺乏全局感知、容易盲目修改代码从而造成破坏性变更的核心痛点。

## 核心价值
- **神经系统与代码地图**：将代码库索引为知识图谱，作为 AI Agent 的“地图”。
- **爆炸半径 (Explosion Radius)**：在修改前通过图谱分析受影响的完整依赖关系和调用链，避免破坏性变更。
- **MCP 集成**：内置 7 大 MCP 工具，支持多种主流语言。
- **零 Token 消耗**：索引阶段（包括向量生成和图谱构建）在本地完成，不消耗大模型 Token。

## 核心功能
*   **代码图谱化**：直接将整个代码库索引为知识图谱，向 AI 展示完整的依赖关系和调用链。
*   **七大内置 MCP 工具**：
    1.  **爆炸半径分析**：分析修改代码可能波及的范围。
    2.  **360度符号视图**：全方位查看符号定义与引用。
    3.  **竞争感知混合搜索**：结合关键词与语义的搜索能力。
    4.  **风险评估工具**：评估代码变更的潜在风险。
    5.  **协调多文件重命名工具**：跨文件的安全重命名。
    6.  **原始图查询工具**：直接进行 Cypher 图查询。
    7.  **全局仓库注册表**：管理多个已索引的仓库。
*   **广泛的多语言支持**：支持 TypeScript, JavaScript, Python, Java, C++, C#, Go, Rust, PHP 等主流语言。
*   **高级项目操作**：辅助重构、GitHub Issues Bug 溯源、PR (Pull Request) Code Review 和风险评估。

## 技术架构
*   **MCP (Model Context Protocol) 协议接入**：作为一个实时的查询引擎，通过 MCP 协议将精准的代码依赖和图谱数据提供给 AI 编程助手。
*   **混合索引机制**：
    *   **基础索引**：生成图数据库、agentx 以及 BM25 关键词索引，速度极快（通常十几秒即可完成）。
    *   **完整（高级）索引**：利用本地 **Hugging Face 嵌入模型**生成语义向量，实现高级语义搜索，不消耗大模型 Token。

## 与同类工具（以 Graphify 为例）的区别

| 对比维度 | GitNexus | Graphify |
| :--- | :--- | :--- |
| **核心定位** | Agent 的“代码地图” | “知识编译器” |
| **专注领域** | 精准的代码结构感知（调用链、爆炸半径等） | 跨模态的语义知识问题处理 |
| **处理对象** | 纯代码文件、符号与调用关系 | 代码、文档、论文、图像、视频等混合模态 |
| **应用场景** | “修改这个函数会影响哪些模块？” | “这段代码对应论文的哪个部分？设计理由？” |

## 安装与配置步骤
1.  **全局安装**：运行官方提供的全局安装命令，通过 `version` 命令确认安装。
2.  **MCP 环境初始化**：运行 `setup` 命令，自动为 Cursor、Claude Code 等工具配置 MCP 协议及 Skill 环境。
3.  **项目代码索引**：
    -   `cd` 进入项目目录。
    -   运行**基础索引**命令：快速生成关键词图数据库。
    -   运行**高级索引**命令：利用本地模型生成语义向量（推荐）。
4.  **启动可视化 WebUI**：运行本地 HTTP 服务器命令，通过浏览器访问 Web 界面直观查看代码节点和依赖关系。
